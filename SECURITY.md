# Security Policy

At EZClaw, we take the security of our users and their data very seriously. Because EZClaw is an autonomous agent capable of physically interacting with your operating system, we have designed it with a security-first and privacy-first mindset.

## Supported Versions

Currently, we provide security updates and patches for the latest major version of EZClaw.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## How We Handle Your Data

EZClaw is built to be a **local-first** application. 

1. **API Keys & Tokens:** Your LLM API keys (OpenAI, Gemini, Anthropic, OpenRouter, etc.), Telegram tokens, and Discord tokens are **never** transmitted to any server owned by us. They are encrypted and stored entirely locally on your machine using `electron-store`.
2. **Chat History & Logs:** Your conversation history and system logs are stored locally on your hard drive. We do not collect telemetry, chat logs, or analytics.
3. **Screenshots:** To perform vision-based tasks, EZClaw captures screenshots. These screenshots are temporarily kept in memory, encoded to base64, sent directly to your chosen LLM provider, and then instantly discarded from history to save memory. They are not saved as files on your disk.

## Security Features Built-in

1. **The Ultimate Kill Switch (`Ctrl + Shift + K`)**: Since EZClaw can take control of your mouse and keyboard, we implemented a global hardware-level kill switch. Pressing `Ctrl+Shift+K` at any time will instantly kill all AI processes and restart the application, returning full control of the PC to you.
2. **Zero Auto-Run Commands**: All file executions and PowerShell scripts generated by the LLM are heavily sandboxed within temporary files that are deleted immediately after execution. 

## Reporting a Vulnerability

If you discover a security vulnerability within EZClaw, please do not disclose it publicly. Instead, we kindly ask that you report it to us directly so we can patch it immediately.

**How to report:**
1. Open an issue on our GitHub repository with the tag `[SECURITY]`.
2. Do NOT include sensitive payloads in public issues. Provide a proof of concept or steps to reproduce using generic data.
3. If it is a critical vulnerability (e.g., Remote Code Execution without user prompt, API key leakage), please send an email to the repository maintainer (check GitHub profile for email) or request a private security advisory on GitHub.

We will acknowledge your report within 48 hours and work with you to understand and resolve the issue as quickly as possible.

## Best Practices for Users

While we build security into the app, autonomous AI comes with inherent risks. Please follow these best practices:
- **Do not run EZClaw with Administrator/Elevated privileges** unless absolutely necessary.
- **Set API Key Limits:** When generating API keys on OpenRouter, OpenAI, or other providers, always set a strict spending limit to prevent accidental API quota exhaustion if the AI gets stuck in a loop.
- **Supervise Initial Runs:** Always supervise the AI when executing complex tasks for the first time. Keep your hand near `Ctrl+Shift+K`.

Thank you for helping keep the EZClaw community secure!
